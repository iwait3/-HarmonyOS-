# HarmonyOS笔记管理应用 - 项目开发指南

## 项目概述

这是一个基于HarmonyOS的现代笔记管理应用，采用ArkTS语言开发，具有以下核心特性：

- 现代化UI设计，符合HarmonyOS设计语言
- 支持文本、图片、文档、音频等多种内容类型
- 强大的搜索功能（标题、标签、内容多维度搜索）
- 批量操作支持
- 撤销删除功能
- 本地数据持久化存储

## 从零开始搭建项目

### 第一步：环境准备

1. **安装DevEco Studio**
   - 下载并安装最新版DevEco Studio
   - 配置HarmonyOS SDK（API版本6.0.1+）

2. **创建新项目**
   - 使用DevEco Studio创建Empty Ability项目
   - 项目模板：Empty Ability
   - 语言：ArkTS
   - API版本：6.0.1(21)

### 第二步：项目配置

1. **配置应用信息** - AppScope/app.json5
```json5
{
  "app": {
    "bundleName": "com.cn.myapplication",
    "vendor": "example",
    "versionCode": 1000000,
    "versionName": "1.0.0",
    "icon": "$media:book",
    "label": "$string:app_name"
  }
}
```

2. **配置模块信息** - entry/src/main/module.json5
```json5
{
  "module": {
    "name": "entry",
    "type": "entry",
    "pages": "$profile:main_pages",
    "abilities": [
      {
        "name": "EntryAbility",
        "srcEntry": "./ets/entryability/EntryAbility.ets",
        "icon": "$media:book",
        "label": "$string:EntryAbility_label",
        "skills": [
          {
            "entities": ["entity.system.home"],
            "actions": ["ohos.want.action.home"]
          }
        ]
      }
    ],
    "requestPermissions": [
      {
        "name": "ohos.permission.READ_IMAGEVIDEO",
        "reason": "$string:read_imagevideo_reason"
      }
    ]
  }
}
```

### 第三步：数据模型设计

创建数据模型文件 entry/src/main/ets/model/NoteData.ts：

```typescript
export enum NoteContentType {
  TEXT = 'text',
  IMAGE = 'image', 
  DOCUMENT = 'document',
  AUDIO = 'audio'
}

export interface NoteContentItem {
  type: NoteContentType;
  content: string;
  metadata?: Record<string, string | number>;
  timestamp: string;
}

export interface NoteData {
  id: string;
  title: string;
  contentItems: NoteContentItem[];
  createTime: string;
  updateTime?: string;
  tags?: string[];
  isPinned?: boolean;
}
```

### 第四步：服务层实现

1. **核心数据服务** - NoteService.ts
   - 使用Preferences进行数据持久化
   - 实现CRUD操作和缓存机制
   - 支持批量操作和搜索功能

2. **文件选择服务** - FilePickerService.ts
   - 处理图片、文档、音频文件选择
   - 文件信息提取和验证

3. **用户交互服务** - ConfirmationService.ts
   - 统一管理确认对话框
   - 提示信息和Toast管理

### 第五步：UI组件开发

1. **主页面组件** - Index.ets
   - 笔记列表展示
   - 搜索和过滤功能
   - 批量操作支持
   - 撤销删除功能

2. **编辑页面组件** - EditNote.ets
   - 笔记创建和编辑
   - 多类型内容管理
   - 文件附件支持
   - 自动保存机制

## 核心功能实现要点

### 1. 数据持久化
- **技术选型**：使用HarmonyOS Preferences API
- **缓存机制**：5秒缓存有效期，提升性能
- **数据格式**：JSON序列化存储

### 2. 状态管理
- **响应式设计**：使用@State装饰器
- **数据流**：单向数据流，状态提升
- **生命周期**：合理使用aboutToAppear/onPageShow

### 3. 文件处理
- **权限管理**：READ_IMAGEVIDEO、READ_DOCUMENT等
- **文件选择**：使用系统文件选择器
- **URI处理**：正确处理文件URI和元数据

### 4. 用户体验优化
- **撤销机制**：UndoManager管理删除操作
- **批量操作**：选择模式切换和批量处理
- **搜索优化**：多维度实时搜索
