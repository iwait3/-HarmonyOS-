# -HarmonyOS-
这是一个基于Spring Boot框架开发的超市收银系统教学项目。该系统模拟了超市的基本运营流程，旨在为Java Web开发初学者提供一个结构清晰、功能完整的全栈实践案例。项目实现了从后台商品管理到前台收银结算的核心业务流程，并提供了完整的数据库设计、后端逻辑与前端交互界面。

**项目介绍**

**1. 项目定位与背景**
该项目是一个面向学习者的教学实践项目，其核心目标是帮助初学者将Spring Boot、数据库操作、Web开发等分散的知识点，通过一个完整的业务场景进行串联与整合。项目选择了“超市收银”这一贴近生活、业务逻辑清晰的场景，使得学习者能够更直观地理解软件功能如何映射到实际需求。

**2. 核心技术栈**
项目采用经典的单体应用架构，技术选型围绕Spring Boot生态展开：

| 技术组件 | 用途说明 |
| :--- | :--- |
| **Spring Boot 3.2.0** | 项目基础框架，提供自动配置、依赖管理等核心能力。 |
| **Spring Data JPA (Hibernate)** | 作为ORM框架，用于简化数据库操作，通过实体类映射数据库表。 |
| **MySQL** | 关系型数据库，用于持久化存储所有业务数据。 |
| **Thymeleaf** | 服务器端模板引擎，用于渲染HTML页面，实现前后端数据绑定。 |
| **Maven** | 项目构建与依赖管理工具。 |

**3. 核心功能模块**
系统主要分为两大功能模块：
*   **后台商品管理模块**：此模块面向系统管理员，提供了对商品信息的全面管理功能。管理员可以对商品进行**增（Create）、删（Delete）、改（Update）、查（Retrieve）** 操作，管理的商品属性包括名称、价格、库存数量和描述信息。所有操作均通过Web界面完成，数据变更实时同步至数据库。
*   **前台收银结算模块**：此模块模拟顾客购物流程。顾客（或收银员）可以在前台页面浏览商品列表，将所需商品加入购物车，并可以调整购物车中商品的数量。确认购买后，系统会执行结算操作，生成唯一的订单号、记录下单时间、计算订单总金额，并创建对应的订单记录及订单明细，完成一次完整的销售交易。

**4. 数据库设计**
项目设计并实现了五张核心数据库表来支撑上述功能：
1.  `admin` 表：存储管理员账号信息。
2.  `product` 表：存储所有商品的基础信息。
3.  `cart` 表：作为临时购物车，存储当前会话中选购的商品及其数量。
4.  `orders` 表：存储订单主信息，如订单号、总价、状态等。
5.  `order_item` 表：存储订单的明细信息，与`orders`表通过外键关联，记录每个订单中包含的具体商品、数量及单价。

**5. 项目成果与特点**
最终实现的项目具有以下特点：
*   **功能闭环**：完整实现了“商品管理 -> 购物车 -> 订单生成”的业务闭环，逻辑自洽。
*   **代码结构清晰**：项目按照实体类（Entity）、数据访问层、业务逻辑层、控制层（Controller）和视图层进行组织，结构分明，便于理解和学习。
*   **开箱即用**：博客提供了从数据库建表脚本、项目依赖配置（`pom.xml`）、应用配置（`application.properties`）到实体类代码的完整素材。学习者只需按照步骤配置数据库和开发环境，即可将项目成功运行起来，并通过直观的Web界面进行功能交互。
*   **教学导向明确**：作为一个教学项目，其代码实现侧重于展示技术栈的基本用法和模块间的协作方式，而非追求极致的性能或复杂的业务设计，非常适合作为初学者入门全栈开发的第一个综合性练手项目。
